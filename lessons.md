# Lessons

1. 用户一旦收口范围，先改计划文档，再动实现。
2. AI Gateway 只管模型网关，不把业务系统接入混进来。
3. 多代理并行时，每个 worker 只碰自己的文件；监工和 reviewer 默认只读。
4. 当前目录不是 git 仓库，进度要落在文档里，不能假装有 git 流程。
5. 用户说“放第二盘”时，先确认是项目目录/数据边界，不默认上升为 Docker 或系统级迁移。
6. Vertex 主链路以真正能执行的后端为准；面板里看得到不等于线上可用。
7. 前端准备直连网关前，先验证浏览器接入契约：CORS、OPTIONS、Authorization 预检，或者同域反代入口。
8. 生产环境模型路由必须显式配置，不能依赖模型名前缀猜 upstream。
9. 阶段名不能混装“后端真值校验”和“跨仓命名对齐”，否则执行层会误判范围。
10. 共享站点 token 只能做站点级审计；如果还想追请求，至少补统一 trace id，不能假装能追到最终用户。
11. 浏览器直连共享站点 token 时，必须显式配来源白名单；不接受全开 CORS。
12. 新 client 要物理隔离，不能继续塞回旧的 provider 路由文件里“顺手重构”。
13. 浏览器契约不能只写 allowed_origins，必须同时写清 prod / preview / localhost 的 origin 与 token 矩阵。
14. 临时兼容入口必须带删除时间点；不写退役阶段，兼容口子最后一定常驻。
15. 分阶段迁移文档必须和运行时真实行为一致；若 1A 仍会触发旧链路，就只能二选一：要么前移到 1A，要么在 1A 明确关闭，不能把旧行为留在运行时再嘴上分期。
16. 预检 `OPTIONS` 不会携带自定义头的真实值；Trace 验收只能要求“预检放行头名，正式请求和上游日志对同一个 id”。
17. “内部专用”不是文案结论，必须靠独立内网入口 / 端口 / 路径做物理隔离；同一个公网入口继续接受旧 token 形式，就不算契约已经收口。
18. 一期 `base_url` 写死且只有单网关时，不提前做 `site_token` 多分桶设计；复杂度要跟真实入口数量走。
